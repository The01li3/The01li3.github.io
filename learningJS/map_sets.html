Maps & Sets

Sets

Sets are a collection of unique values, think array, but none the same.
They will return "objects" when typeof is used.
They have no keys, just a list.
Create with: 

const s = new Set([x,y,z])

Use a for...or loop to go through them all.

Method Desc
s.add(x) Adds a value to the set
s.has(x) Checks if a value exists
s.foreach(myFunction) Loops through the set calling myFunction every time.
s.entries Returns an Iterator with the [value,value] pairs from a set
s.clear Empties the set
s.values() s.keys() Returns an Iterator with the values in a Set
s.delete Deletes an element from the set
s.size Returns the number of elements

Maps

Maps hold key-value pairs, the key can be any datatype.
They will return "objects" when typeof is used.
Create with: 

const m = new Map([x,a],[y,b],[z,c])

Use a for...or loop to go through them all.

Method Desc
m.set(x,a) Adds a value to the map
m.get(x) Gets the value, with a key of x
m.has(x) Checks if the map has x as a key
m.forEach(myFunction) Loops through the map calling myFunction every time.
m.clear Empties the map
m.entries() Returns an Iterator with the [value,value] pairs from a map
m.delete() Deletes an element from the map
m.keys() Returns an iterator object with the keys in a map:
m.size Returns the number of elements

You can use objects as keys, to then make finding it use a value

const apples = {name: 'Apples'};
const fruits = new Map();
fruits.set(apples, 500);
fruits.get(apples); //not fruits.get("apples"); this is undefined.

Differences

Object Maps
Not directly iterable Directly iterable
Do not have a size property Have a size property
Keys must be Strings (or Symbols) Keys can be any datatype
Keys are not well ordered Keys are ordered by insertion
Have default keys Do not have default keys
