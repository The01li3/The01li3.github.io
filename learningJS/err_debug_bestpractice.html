Errors, Debugging, Best Practice

Errors

Can use a try... catch... finally block.
try will run the code inside it.
catch will run if the try fails with either a system error message or a throw message.
finally gets run regardless of the try/catch result.

In catch(e) the e is the error object, it consists of a name, and a message, display it, console.log it, whatever, its fairly useful to see what went wrong.

Debugging

F12 really helpful, can use it to break at certain points, then do a step-over to see whats causing the error.
Console.log() will also likely get used a lot to see certain variables, and what XYZ happens.
There's also a debugger; command, that acts the same way as putting in a break point in the debugger.

Strict

"use strict; will make the code run in strict mode.
It can also be used inside a function for block scoping.
Only works if at the begging of the script or function.
Makes it so that the code is less lenient on bad syntax, need to declare variables, less likely to mistype a variable, cannot delete a variable etc.

Styling

Normally use Hungarian notation in camelCase:
vVariable, bBool, cCount, arrArray... etc.
Apart from global variables that are often full caps, with an underscore INVOICETYPE_INVOICE
css and html classes with hyphens btn-primary-eg

Spaces tend to appear around operators 1 + 1 = 2 and after commas name = ["Alice", "Bob", "Eve"]

W3Schools says to use 2 spaces instead of tabs based off the fact editors can interpret tabs differently... I've always just used tabs.

Use lowercase file names, servers can look at them different so consistent is lowercase.

Best Practice

Try to avoid using global variables.
Declare variables, preferably as high up as possible.
Initialise variables where possible, helps recognise what the variable type is.
Objects and arrays declared with const.
Dont use new Object(); just declare them as empty "", 0, [], {}... etc.
=== where possible, saves something like 0=="" OR 1==true //true and leading to complications.
Function undefined variables can break it, can assign default values: myFunction(a=1, b=1){} to fill in if the variable is undefined.
Switches end with defaults in case there's a case statement missing for XYZ value.

Common Mistakes

== is comparison = is assign. if (x = 10) will be true.
Switch statements use ===.
+ ... can be strings or numbers.
floats are odd 0.1 + 0.2 = 0.30000000000000004.
\ to break up js string variables.
Undefined is not null.
JS starts at 0.

Performance

Try to avoid assign statements in a for loop to make the loop faster:

let l = arr.length;
for (let i = 0; i < l; i++) {}

Searching for HTML can also be slow so the below is better:

const obj = document.getElementById("demo");
obj.innerHTML = "Hello";

However adversely, don't create variable when you can get away without

let fullName = firstName + " " + lastName;
document.getElementById("demo").innerHTML = fullName;

document.getElementById("demo").innerHTML = firstName + " " + lastName;

Reserved Words

Some words you cannot use, use Hungarian notation and you'll be fine.
