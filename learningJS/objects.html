Objects

Intro

Object properties, are the value keys stored inside the object car.name = "fiat";
Object methods are defined with brackets car.start();
They are defined with {} and can have values added later (functions need prototype to be added to the main section), or they can have values assigned at the start: 

const car = {type:"Fiat",  //at start
    model:"500", 
    color:"white",
    fullName: function(){
        return this.type + " " + this.model;
    }
}; 

const car = {}; //added later
car.type = "Fiat";
car.model = "500";
car.color = "white";
car.prototype.fullName = function(){
    return this.type + " " + this.model;
}

Can access these properties with either car.type, car["type"] or car[x] useful for accessing dynamically.
Objects can store functions such as "fullName" above.
Objects are mutable, so const x = car; will make x BE car, not a copy of, a direct line to.

Definition

Defined with {}, you can then add properties if need be, can also define it with properties as above, or can use contructor functions:

function Person(first, last, age, eye) {
    this.firstName = first;
    this.lastName = last;
    this.age = age;
    this.eyeColor = eye;
}

const myMother = new Person("Alice", "Doe", "47", "green");
const myFather = new Person("John", "Doe", 50, "blue");

If a function needs to be added to Person, rather than each object of Mother or Father, you need the prototype constructor.

Properties

Can access properties with .x, ["x"] or [variable].
New properties can be added the same way with =.
delete ... will delete property and value, make it undefined.
You can contain objects in objects with the same {} notation, then accessed with x.y.z; etc. as above.

There are some properties that have a longform, if you want to change characteristics.


Object.defineProperty(person,"age",{value:"47"}); would be the same as person.age = "47";.
But this means you can Object.defineProperty(person, "age", {writable:false}); to make it read-only.

Object.getOwnPropertyNames(object); and Object.keys(person); will produce a list of all/enumerable properties respectively.

Object.defineProperty() can be used to add Getters and Setters.

Methods

Objects methods are actions performed on the objects.
They have a function, stored as a property value.
You can use this in the function to refer to the objects properties.
These functions can then be accessed with the property name, and (); same as standard function calls.
If you try to access the property without the brackets it'll return the function, not the data.

Object.assign(object1, object2) will overwrite object1, with object 2 for any matching properties, or add mismatching ones.
Object.entries(object1) will output a comma separated list of keys and values. for (let [fruit, value] of Object.entries(fruits)) being a good way to loop and use the key and value in the loop.
Object.values(object1) works the same way, but only the values.
However can also use for (let x in person) and loop person[x] for just the values.

Prototype

Prototype is the name of the initial object rather than whats being created:

function Person(first, last, age, eyeColor) { //prototype
    this.firstName = first;
    this.lastName = last;
    this.age = age;
    this.eyeColor = eyeColor;
}

const myFather = new Person("John", "Doe", 50, "blue");// objects
const myMother = new Person("Sally", "Rally", 48, "green");

So wanting to add "nationality" to this object you either have to use person.prototype.nationality or just add it to the initial declaration.
Just don't edit JS's prototypes, only your own.

Protection

const is the best way to try and protect an object/array to stop reassigning.
However there are a few object protection methods too.

Examples with "car" as the object

Method Desc
Object.preventExtensions(car); Prevents adding any more properties to the object
Object.seal(car) Prevents additions or deletions of properties
Object.freeze(car) Prevents any changes to the object
