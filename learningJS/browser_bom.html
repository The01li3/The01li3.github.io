Browser BOM

Intro

Browser Object Model, name of the collective group of methods and properties of BOM.
Window object is something that's supported by all browsers. all functions, objects variable etc are part of the window object.
Because it's and object this means it had properties:

  • window.innerWidth
  • window.innerHeight

It also has methods:

  • window.open/close()
  • window.moveTo()
  • window.resizeTo()

Screen

window.screen object contains information about the users screen.

  • screen.width //actual screens width.
  • screen.height // actual screens height.
  • screen.availWidth // available to show, removes taskbar etc
  • screen.availHeight // available to show, removes taskbar etc
  • screen.colorDepth // returns the number of bits used to display one color.
  • screen.pixelDepth // returns the pixel depth of the screen.

Location

window.location contains the current pages URL, and can direct somewhere else.
Only really useful to direct with href as far as I can see, although some might be needed for security etc.

History

history.back() and history.forward() is the same as clicking back and forward in the browser.

Navigator

Will contain information about the browser.
Not super useful, but possible to use to make sure XYZ in enabled, or make XYZ browser look different.

Popups

We have 3 types of popup available to us, with the ability to use \n for line breaks:

  • alert("hello");
  • confirm("hello?"); //returns true/false depending on ok.cancel
  • let demo = prompt("hello?","DEFAULT TEXT") //demo will be null/"" if cancelled, or filled in.

Timing

setTimeout(function,milliseconds), will run that function after X milliseconds.
setInterval(function,milliseconds), will call that function continuously every X milliseconds until clearInterval() is called.
While they can be used, promises are just better.

Cookies

Cookies are data, stored in small text files, on your computer.

document.cookie = "username=John Doe; expires=Thu, 18 Dec 2013 12:00:00 UTC; path=/";

Stores in pairs x=y with ; to separate them here we have a value, expiry date, and a path parameter, defaults to current page. Cookies with the same value can be overwritten to change it, and to delete them, just set the date to the past. 

function setCookie(cname,cvalue,exdays) {
    const d = new Date();
    d.setTime(d.getTime() + (exdays*24*60*60*1000));
    let expires = "expires=" + d.toUTCString();
    document.cookie = cname + "=" + cvalue + ";" + expires + ";path=/";
}

function getCookie(cname) {
    let name = cname + "=";
    let decodedCookie = decodeURIComponent(document.cookie);
    let ca = decodedCookie.split(';');
    for(let i = 0; i < ca.length; i++) {
        let c = ca[i];
        while (c.charAt(0) == ' ') {
        c = c.substring(1);
        }
        if (c.indexOf(name) == 0) {
        return c.substring(name.length, c.length);
        }
    }
    return "";
}

function checkCookie() {
    let user = getCookie("username");
    if (user != "") {
        alert("Welcome again " + user);
    } else {
        user = prompt("Please enter your name:","");
        if (user != "" && user != null) {
            setCookie("username", user, 30);
        }
    }
}
