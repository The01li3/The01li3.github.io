Numbers

One number type, can be written with or without decimals, you can use 123e5 type numbers.
They are 64bit, so precision can become an issue: 

0.2 + 0.1 =

However JS will try to convert to numeric with * / -.

"6" * "2" =

NaN Not a Number can check with isNaN().
Infinity exists if the number is outside the scope or /0 (should really be undefined.) -Infinity can also exist.
Numbers can be created as objects... dont, it just makes it slower, JS object comparing always returns false.
Number conversion can be odd, but its base10 by default.

BigInt

Integers only have 15 digits precision, after that have to use a BigInt

typeof(BigInt(1234567890123456789012345))   //bigint
typeof(1234567890123456789012345n)          //bigint
typeof(1234567890123456789012345)           //number

Number.isSafeInteger(x) can be used to see what type it might need to be.
Can use the normal operators you can use with numbers, but have to be with other BigInts
BigInt cannot have decimal points.

Number Methods

Quick glance at some methods used with numbers

Method Desc
toString() Returns a number as a string
toExponential() Returns a number written in exponential notation
toFixed() Returns a number written with a number of decimals
toPrecision() Returns a number written with a specified length
valueOf() Returns a number as a number

Number methods can be found here.
Number() will return a number, works with dates for milliseconds too. Has to be able to be number, i.e. "10 33" is invalid.
parseInt() and parseFloat() will return Int and Float, taking the first one, i.e. "10 66" will be "10".
Number.IsInteger vs Number.IsSafeInteger, "safe" will fit inside a number, so a little more precise than not "safe".

Number Properties

Not going to be used all that much, but for future reference

Property Desc
