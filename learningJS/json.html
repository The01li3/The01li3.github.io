JSON

Intro

JavaScript Object Notation

We know JSON, it is what Ajax normally returns.

'{"name":"John", "age":30, "car":null}'

Defines an object with 3 properties: Name, Age and Car.
Can then use obj.name or ob["name"] to access it, reassign it.

Values must be strings, numbers, objects, arrays, boolean or null.
Values cannot be functions, dates, undefined.

You can use JSON.parse(jsonInHere) to make the JSON a JS Object.
Since dates are not allowed in JSON, you write it as a string, then can convert it as needed.

JSON.stringify(objInHere) will convert the obj to a string in order to send it to the webserver if needed.
It will also convert dates to strings, so they can be used in JSONs 

myObj = {"name":"John", "age":30, "car":null}; //JSON literal

const myJSON = '{"name":"John", "age":30, "car":null}';
const myObj = JSON.parse(myJSON); //JSON string, apparently this is the normal way

let text = "";
for (const x in myObj) {
    text += x + ", "; //Name
    text += myObj[x] + ", "; //Value
}

You can use it with a PHP server, but its fairly damn niche and can be googled if need be.
Data can be added into HTML as needed, template literals work well, or can just assign to innerHTML.
JSONP is a method for sending JSON data without worrying about cross-domain issues.
