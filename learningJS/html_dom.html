HTML DOM

Intro

HTML Document Object Model.
JS is able to change the elements of an HTML format because of this.
The HTML DOM is a standard for how to get, change, add, or delete HTML elements. Here has a small example of possible changes.

Examples

Main

Main one to be used is going to be document.getElementById, you can also do TagName, and ClassName.
innerHTML, attribute, style.property, can all be changed.
If you need something to happen to any HTML there's probably a way to do it, and very googleable.
You can assign a variable the method const x = document.getElementById("main"); to find all the elements that have a main id, then const y = x.getElementsByTagName("p"); to find all p tags inside that main.
const x = document.querySelectorAll("p.intro"); will find all p tags that have a class of "intro".
x.src = image.jpg can be used to change the image.

Forms

onsubmit="validateForm()" for when a form is submitted... shock!
required will make the form input have to be filled in.
You can also add validation, min/max numbers etc.

CSS

document.getElementById("p2").style.color = "blue"; self explanatory.
Can be done on click and such.
document.getElementById("animate").style.top, you get the idea.
id = setInterval(<fnMyFunction>,delay) will run the function every X milliseconds, until it reaches clearInterval(id).
Which means the above can be used for animations to move things across the screen.

Event Listener

.addEventListener("click",fnMyFunction) it doesn't use "onclick" like previous.
You can remove these with .removeEventListener().
Means you can add multiple events to a single DOM Object. Doesn't have to be HTML i.e. the window object.
Takes it out of the HTML, makes it just a little easier to read.

There's a third input parameter available to us for event bubbling/event capturing.
Imagine a p element inside a div element, the user then clicks on the p, both having an "onclick".
Bubbling will do inner out.
Capturing will out outer in.
FALSE is default, uses "bubbling". TRUE will then use "capturing".

Navigation

You can move around HTML nodes with some properties.

  • parentNode
  • childNodes[nodeNumber]
  • first/lastChild
  • next/previousSibling

innerHTML is the same as nodeValue

myTitle = document.getElementById("demo").innerHTML;
myTitle = document.getElementById("demo").firstChild.nodeValue;
myTitle = document.getElementById("demo").childNodes[0].nodeValue;

Will all do the same thing.

You can cycle through element tags as they get saved in and look like arrays, but are not. Accessible with examplePTag[x].
Means examplePTag.length is a property you can cycle through a for loop to change them all.

Node Manipulation

You can add new elements with:

const para = document.createElement("p");
const node = document.createTextNode("This is new.");
para.appendChild(node);

const element = document.getElementById("demo");
element.appendChild(para);

Creating the element and calling it "para", creating a text node, then adding it to para. Para can then get appended to the end of the demo div.
Can also do element.insertBefore(para,<element to insert before>) element.remove() can also be used to remove the element. mainDiv.replaceChild(new,old) will replace, where mainDiv is the div that the nodes are under.
