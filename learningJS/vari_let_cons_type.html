Variables, Let, Const & Data Types

Variables

var isn't really use any more, stopped 2015, so use let where possible:

  1. Always declare variables.
  2. Always use const if the value should not be changed.
  3. Always use const if the type should not be changed (Arrays and Objects).
  4. Only use let if you can't use const.
  5. Only use var if you MUST support old browsers.
const cantReassignMe = "Nope."; //used on objects to change properties, but not headings.
let definedWithinScope;
var imAnInt = 2,
    imAChar = "2",
    imCreatedUndefined,
    imAChar; //redeclaring doesn't clear

var $ = 1; //valid but not always used, mainly for library uses (jQuery etc)
var _ = 2; //normally used for hidden variables

Let

var doesn't have a block scope so can be used anywhere.
let has a block scope, so only used within the brackets.
let cannot be redeclared, but you can with var.

Scope Redeclare Reassign Hoisted Binds this
var No Yes Yes Yes Yes
let Yes No Yes No No
const Yes No No No No

Variables with var are "hoisted" meaning they can be initialised anywhere and the code will "hoist" the declaration to the top.

Const

const cannot be reassigned, and must be assigned when declared.
So const should be used for Arrays, Objects, Functions, RegExp.
const is not a constant, value, but a constant reference to a value. So can be used to change elements in an array/object.
See "let" for table on whats re-declarable etc.

Data Type

Overview

Js has 8 datatypes

  • String "blue"
  • Number 8.34
  • Bigint 9999999999999999999999999
  • Boolean TRUE
  • Undefined
  • Null
  • Symbol
  • Object

Arrays

Arrays seem odd, defined with const so you cannot full change, but you can part change

const vArray = ["Alice", "Bob", "Eve"]  
const cars[0]= "Charlie"; cars[1]= "Dave"; cars[2]= "AnotherName";

^ this is all ok, and doesn't go against the const, however:

vArray = ["Charlie", "Dave", "AnotherName"]

Will error.

Can convert to string with cars.toString().
And print with document.getElementById("demo").innerHTML = cars;
Fairly standard .length and [0] etc, arrays use number pointer, objects use string.
You can store objects and functions in arrays cars[0] = Date.Now() etc.
